Laravel Queues Masterclass: Background Jobs That Scale

Master Laravel queues from job creation to production monitoring.

Laravel queues are essential for building responsive applications. Move time-consuming tasks like sending emails, processing images, or generating reports to the background, and respond to users instantly.

Create a Job

php artisan make:job ProcessPayment

// app/Jobs/ProcessPayment.php
namespace App\Jobs;

use App\Models\Order;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;
use Illuminate\Queue\SerializesModels;

class ProcessPayment implements ShouldQueue
{
    use Dispatchable, Queueable, SerializesModels;
    
    public $tries = 3;
    public $timeout = 120;
    public $backoff = [30, 60, 120];
    
    public function __construct(public Order $order) {}
    
    public function handle(): void
    {
        $paymentService = app(PaymentService::class);
        $paymentService->process($this->order);
        
        $this->order->update(['status' => 'paid']);
    }
    
    public function failed(\Throwable $exception): void
    {
        Log::error('Payment failed', [
            'order_id' => $this->order->id,
            'error' => $exception->getMessage()
        ]);
        
        $this->order->update(['status' => 'failed']);
    }
}

Dispatch Jobs

// Dispatch immediately
ProcessPayment::dispatch($order);

// Dispatch with delay
ProcessPayment::dispatch($order)->delay(now()->addMinutes(5));

// Dispatch to specific queue
ProcessPayment::dispatch($order)->onQueue('payments');

// Dispatch conditionally
ProcessPayment::dispatchIf($order->amount > 1000, $order);

// Chain jobs
Bus::chain([
    new ProcessPayment($order),
    new SendInvoice($order),
    new NotifyCustomer($order),
])->dispatch();

Run Workers in Production

# Supervisor configuration for production
[program:laravel-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/artisan queue:work redis --sleep=3 --tries=3
autostart=true
autorestart=true
numprocs=8
redirect_stderr=true
stdout_logfile=/var/log/worker.log

Key Takeaways

  • Retry logic: Configure tries and backoff for resilience
  • Job chaining: Execute jobs in sequence
  • Supervisor: Use for production worker management
  • Horizon: Use Laravel Horizon for monitoring
