Integrating Stripe Payments: A Developer's Complete Guide

Accept payments securely with Stripe Checkout, subscriptions, and webhooks.

Payment integration can be complex, but Stripe makes it manageable. After implementing Stripe in multiple production applications, here's everything you need to know.

One-Time Payments with Checkout

// Create Checkout Session
app.post('/create-checkout', async (req, res) => {
  const session = await stripe.checkout.sessions.create({
    payment_method_types: ['card'],
    line_items: [{
      price_data: {
        currency: 'usd',
        product_data: { name: product.name },
        unit_amount: product.price * 100,
      },
      quantity: 1,
    }],
    mode: 'payment',
    success_url: `${origin}/success?session_id={CHECKOUT_SESSION_ID}`,
    cancel_url: `${origin}/cancel`,
  });
  
  res.json({ url: session.url });
});

Subscription Payments

const session = await stripe.checkout.sessions.create({
  customer: user.stripeCustomerId,
  line_items: [{
    price: 'price_xxxxxxxxxxxxx', // From Stripe Dashboard
    quantity: 1,
  }],
  mode: 'subscription',
  success_url: `${baseUrl}/subscription/success`,
  cancel_url: `${baseUrl}/subscription/cancel`,
});

Webhooks - The Critical Part

Never trust client-side confirmations. Use webhooks:

app.post('/webhooks/stripe', express.raw({ type: 'application/json' }), 
  async (req, res) => {
    const event = stripe.webhooks.constructEvent(
      req.body,
      req.headers['stripe-signature'],
      process.env.STRIPE_WEBHOOK_SECRET
    );
    
    switch (event.type) {
      case 'checkout.session.completed':
        await handleCheckoutComplete(event.data.object);
        break;
      case 'invoice.paid':
        await handleInvoicePaid(event.data.object);
        break;
    }
    
    res.json({ received: true });
  }
);

Test Cards

  • 4242 4242 4242 4242 - Successful payment
  • 4000 0000 0000 0002 - Card declined
  • 4000 0000 0000 9995 - Insufficient funds
  • 4000 0027 6000 3184 - Requires 3D Secure

Security Checklist

  • ✅ Never log full card numbers
  • ✅ Use webhook signature verification
  • ✅ Store Stripe IDs, not card details
  • ✅ Use HTTPS everywhere
  • ✅ Implement idempotency for retries

Conclusion

Stripe's documentation is excellent, but real-world integration has nuances. Always test thoroughly and never trust client-side payment confirmations.
