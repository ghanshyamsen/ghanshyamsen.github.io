MySQL Query Optimization: 10 Techniques That Actually Work

Transform slow queries into lightning-fast operations with these proven techniques.

Database performance can make or break your application. I've optimized queries that went from 30 seconds to 30 milliseconds using these techniques.

1. Use EXPLAIN to Understand Queries

EXPLAIN SELECT users.name, COUNT(orders.id) as order_count
FROM users
LEFT JOIN orders ON users.id = orders.user_id
WHERE users.created_at > '2024-01-01'
GROUP BY users.id;

-- Look for: type (ALL=bad, ref=good), key, rows, Extra

2. Index the Right Columns

-- Index columns used in WHERE, JOIN, and ORDER BY
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_orders_user_id ON orders(user_id);

-- Composite indexes for multi-column queries
CREATE INDEX idx_orders_status_created ON orders(status, created_at);

3. Avoid SELECT *

-- Bad
SELECT * FROM orders WHERE user_id = 123;

-- Good
SELECT id, status, total, created_at 
FROM orders WHERE user_id = 123;

4. Use Keyset Pagination

-- Offset pagination (slow with high offsets)
SELECT * FROM products LIMIT 20 OFFSET 10000;

-- Keyset pagination (consistent performance)
SELECT * FROM products 
WHERE id > 10000 
ORDER BY id LIMIT 20;

5. Optimize JOINs

-- Ensure joined columns are indexed
CREATE INDEX idx_orders_user_id ON orders(user_id);

SELECT o.*, u.name
FROM orders o
INNER JOIN users u ON o.user_id = u.id
WHERE o.status = 'completed';

Quick Wins Summary

  • EXPLAIN - Understand query execution
  • Indexes - Speed up WHERE, JOIN, ORDER BY
  • Select specific columns - Reduce I/O
  • Keyset pagination - Consistent performance
  • Batch operations - Reduce round trips
  • Monitor slow queries - Enable slow query log

Conclusion

Query optimization is an iterative process. Start with EXPLAIN, add appropriate indexes, and measure the impact. Small changes in how you write queries can lead to dramatic performance improvements.
