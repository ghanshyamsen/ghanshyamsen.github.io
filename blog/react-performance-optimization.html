React Performance Optimization: From Good to Great

Build lightning-fast React applications with these proven optimization techniques.

A fast application isn't just about user experience—it's about business results. Users abandon slow apps, and search engines penalize them. Here's how to identify and fix performance bottlenecks in your React applications.

1. Understanding Re-renders

React re-renders a component when:

  • Its state changes
  • Its props change
  • Its parent re-renders
  • Context it consumes changes

2. React.memo for Pure Components

// With memoization - only re-renders when user prop changes
const UserCard = React.memo(({ user }) => {
  return (
    <div className="user-card">
      <h3>{user.name}</h3>
      <p>{user.email}</p>
    </div>
  );
});

3. useMemo for Expensive Calculations

function Dashboard({ transactions }) {
  const stats = useMemo(() => {
    return calculateStats(transactions);
  }, [transactions]);
  
  return <StatsDisplay stats={stats} />;
}

4. useCallback for Stable References

function ParentComponent() {
  const [count, setCount] = useState(0);
  
  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []);
  
  return <ChildComponent onClick={handleClick} />;
}

5. Code Splitting with React.lazy

import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./Dashboard'));

function App() {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
      </Routes>
    </Suspense>
  );
}

6. Virtualize Long Lists

import { FixedSizeList } from 'react-window';

function UserList({ users }) {
  const Row = ({ index, style }) => (
    <div style={style}>{users[index].name}</div>
  );

  return (
    <FixedSizeList
      height={400}
      itemCount={users.length}
      itemSize={50}
      width="100%"
    >
      {Row}
    </FixedSizeList>
  );
}

Key Optimization Techniques

  • React.memo - Prevent unnecessary re-renders
  • useMemo - Cache expensive calculations
  • useCallback - Stable function references
  • Code splitting - Load only what's needed
  • Virtualization - Render only visible items
  • React DevTools Profiler - Identify bottlenecks

Conclusion

Performance optimization is about making informed decisions. Don't optimize prematurely—measure first, then optimize the bottlenecks. These techniques should be part of your toolkit, applied where they make a real difference.
