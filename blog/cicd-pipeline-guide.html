CI/CD Pipeline Setup: From Zero to Production Deploy

Automate testing, building, and deployment with GitHub Actions.

Manual deployments are error-prone, stressful, and slow. A proper CI/CD pipeline automates testing, building, and deployment, letting you ship with confidence.

What We're Building

  1. Run on every push and pull request
  2. Install dependencies and run linting
  3. Execute test suite
  4. Build the application
  5. Deploy to production on main branch

Basic GitHub Actions Workflow

name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run linting
        run: npm run lint
      
      - name: Run tests
        run: npm test

Deploy Job

  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to server
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /var/www/myapp
            git pull origin main
            npm ci --production
            pm2 restart myapp

Best Practices

  • Cache dependencies - Speeds up builds
  • Fail fast - Run linting before tests
  • Use matrix builds - Test multiple versions
  • Keep secrets secure - Never log them
  • Add status badges - Show build status in README

Conclusion

CI/CD transforms how teams ship software. Start simple, then add complexity as needed. The investment in automation pays dividends in reliability, speed, and developer happiness.
