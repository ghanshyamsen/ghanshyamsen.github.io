Scaling WebSockets with Redis Pub/Sub: Multi-Server Setup

Scale your Socket.io application across multiple servers using Redis adapter.

When your WebSocket application grows beyond a single server, you need a way to synchronize messages across all instances. Redis Pub/Sub is the solution, and Socket.io makes it incredibly easy to implement.

The Problem

Without Redis, each Socket.io server maintains its own list of connected clients. If User A is connected to Server 1 and User B to Server 2, messages from A won't reach B. Redis solves this by acting as a message broker between all servers.

Install Dependencies

npm install socket.io @socket.io/redis-adapter redis

Configure Socket.io with Redis

const { Server } = require('socket.io');
const { createAdapter } = require('@socket.io/redis-adapter');
const { createClient } = require('redis');
const http = require('http');

const server = http.createServer();
const io = new Server(server, {
  cors: { origin: '*' }
});

// Create Redis clients
const pubClient = createClient({ url: process.env.REDIS_URL });
const subClient = pubClient.duplicate();

// Connect and set up adapter
Promise.all([pubClient.connect(), subClient.connect()]).then(() => {
  io.adapter(createAdapter(pubClient, subClient));
  console.log('Redis adapter connected');
  
  server.listen(3000, () => {
    console.log('Socket.io server running on port 3000');
  });
});

io.on('connection', (socket) => {
  console.log('User connected:', socket.id);
  
  socket.on('join-room', (roomId) => {
    socket.join(roomId);
  });
  
  socket.on('message', (data) => {
    // This broadcasts to ALL servers via Redis
    io.to(data.roomId).emit('message', data);
  });
});

Nginx Load Balancer with Sticky Sessions

upstream socket_servers {
    ip_hash;  # Sticky sessions based on client IP
    server 127.0.0.1:3001;
    server 127.0.0.1:3002;
    server 127.0.0.1:3003;
}

server {
    listen 80;
    
    location /socket.io/ {
        proxy_pass http://socket_servers;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

Key Takeaways

  • Redis Adapter: Synchronizes events across all server instances
  • Sticky Sessions: Required for WebSocket handshake to work
  • Horizontal Scaling: Add more servers as needed
  • Redis Cluster: Use Redis Cluster for high availability
