Secure File Uploads to AWS S3: Pre-signed URLs Explained

Learn how to implement secure, direct-to-S3 file uploads using pre-signed URLs.

Uploading files through your server is inefficient and expensive. Pre-signed URLs allow clients to upload directly to S3, reducing server load and improving performance dramatically. In this guide, I'll show you how to implement this pattern securely.

Why Pre-signed URLs?

  • Reduced server load: Files go directly to S3, bypassing your server
  • Better performance: No server bottleneck for large files
  • Cost savings: Less bandwidth and compute usage
  • Scalability: Handle unlimited concurrent uploads

Generate Pre-signed URL in Laravel

use Illuminate\Support\Facades\Storage;
use Aws\S3\S3Client;

public function getUploadUrl(Request $request)
{
    $request->validate([
        'filename' => 'required|string|max:255',
        'content_type' => 'required|string'
    ]);
    
    $filename = $request->input('filename');
    $contentType = $request->input('content_type');
    $path = 'uploads/' . uniqid() . '-' . $filename;
    
    $client = Storage::disk('s3')->getClient();
    
    $cmd = $client->getCommand('PutObject', [
        'Bucket' => config('filesystems.disks.s3.bucket'),
        'Key' => $path,
        'ContentType' => $contentType,
    ]);
    
    $presignedRequest = $client->createPresignedRequest($cmd, '+5 minutes');
    
    return response()->json([
        'upload_url' => (string) $presignedRequest->getUri(),
        'file_path' => $path,
        'expires_in' => 300
    ]);
}

Frontend Upload with Progress

async function uploadFile(file, onProgress) {
  // Step 1: Get pre-signed URL from your server
  const response = await fetch('/api/upload-url', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      filename: file.name,
      content_type: file.type
    })
  });
  
  const { upload_url, file_path } = await response.json();
  
  // Step 2: Upload directly to S3
  await new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open('PUT', upload_url);
    xhr.setRequestHeader('Content-Type', file.type);
    
    xhr.upload.onprogress = (e) => {
      if (e.lengthComputable) {
        onProgress(Math.round((e.loaded / e.total) * 100));
      }
    };
    
    xhr.onload = () => xhr.status === 200 ? resolve() : reject();
    xhr.onerror = reject;
    xhr.send(file);
  });
  
  return file_path;
}

Security Best Practices

  • Short expiration: Keep URLs valid for 5-15 minutes max
  • Validate file type: Check content type before generating URL
  • Limit file size: Set max size in S3 bucket policy
  • Use unique paths: Prevent overwrites with UUIDs

Conclusion

Pre-signed URLs are essential for any application handling file uploads at scale. By offloading uploads directly to S3, you reduce server costs, improve performance, and create a better user experience with progress tracking.
