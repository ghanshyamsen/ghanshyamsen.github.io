Docker for PHP Developers: Complete Local Setup Guide

Learn how to containerize your PHP applications with Docker, set up development environments, and deploy to production seamlessly.

Docker has revolutionized how we develop and deploy applications. For PHP developers, Docker eliminates the "works on my machine" problem and provides consistent environments from development to production. In this comprehensive guide, I'll show you how to set up a complete Docker environment for PHP development.

Why Docker for PHP Development?

Before diving into the setup, let's understand why Docker is a game-changer for PHP developers:

  • Consistency: Same environment across all team members and servers
  • Isolation: Multiple projects with different PHP versions without conflicts
  • Easy onboarding: New developers can start with a single command
  • Production parity: Development environment matches production exactly

1. Basic Docker Setup for PHP

Let's start with a simple Dockerfile for a PHP application. This is the foundation of your containerized setup.

# Dockerfile
FROM php:8.3-fpm

# Install system dependencies
RUN apt-get update && apt-get install -y \
    git \
    curl \
    libpng-dev \
    libonig-dev \
    libxml2-dev \
    zip \
    unzip

# Clear cache
RUN apt-get clean && rm -rf /var/lib/apt/lists/*

# Install PHP extensions
RUN docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd

# Get latest Composer
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# Set working directory
WORKDIR /var/www

# Copy application files
COPY . /var/www

# Install dependencies
RUN composer install --no-interaction --optimize-autoloader

# Expose port 9000 and start php-fpm server
EXPOSE 9000
CMD ["php-fpm"]

2. Docker Compose for Complete Stack

Docker Compose allows you to define and run multi-container applications. Here's a complete setup with PHP, Nginx, MySQL, and Redis:

# docker-compose.yml
version: '3.8'

services:
  # PHP Service
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: php-app
    restart: unless-stopped
    working_dir: /var/www
    volumes:
      - ./:/var/www
      - ./docker/php/local.ini:/usr/local/etc/php/conf.d/local.ini
    networks:
      - app-network

  # Nginx Service
  nginx:
    image: nginx:alpine
    container_name: nginx
    restart: unless-stopped
    ports:
      - "8000:80"
    volumes:
      - ./:/var/www
      - ./docker/nginx/conf.d:/etc/nginx/conf.d
    networks:
      - app-network

  # MySQL Service
  db:
    image: mysql:8.0
    container_name: mysql
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_USER: ${DB_USERNAME}
    volumes:
      - dbdata:/var/lib/mysql
    ports:
      - "3306:3306"
    networks:
      - app-network

  # Redis Service
  redis:
    image: redis:alpine
    container_name: redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  dbdata:
    driver: local

3. Nginx Configuration

Create a proper Nginx configuration for your PHP application:

# docker/nginx/conf.d/app.conf
server {
    listen 80;
    index index.php index.html;
    error_log  /var/log/nginx/error.log;
    access_log /var/log/nginx/access.log;
    root /var/www/public;

    location ~ \.php$ {
        try_files $uri =404;
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        fastcgi_pass app:9000;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }

    location / {
        try_files $uri $uri/ /index.php?$query_string;
        gzip_static on;
    }
}

4. PHP Configuration

Customize PHP settings for development:

# docker/php/local.ini
upload_max_filesize=40M
post_max_size=40M
memory_limit=256M
max_execution_time=600
max_input_time=600

5. Running Your Docker Environment

With everything set up, you can now run your application:

# Build and start containers
docker-compose up -d

# View running containers
docker-compose ps

# View logs
docker-compose logs -f app

# Execute commands in container
docker-compose exec app php artisan migrate

# Stop containers
docker-compose down

# Stop and remove volumes
docker-compose down -v

6. Development Workflow

Here are common commands you'll use daily:

# Install Composer dependencies
docker-compose exec app composer install

# Run migrations
docker-compose exec app php artisan migrate

# Clear cache
docker-compose exec app php artisan cache:clear

# Run tests
docker-compose exec app php artisan test

# Access MySQL
docker-compose exec db mysql -u root -p

# Access container shell
docker-compose exec app bash

7. Production Optimization

For production, create a separate Dockerfile with optimizations:

# Dockerfile.prod
FROM php:8.3-fpm as base

RUN apt-get update && apt-get install -y \
    git curl libpng-dev libonig-dev libxml2-dev zip unzip

RUN docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd opcache

# OPcache configuration
RUN echo "opcache.enable=1" >> /usr/local/etc/php/conf.d/opcache.ini \
    && echo "opcache.memory_consumption=256" >> /usr/local/etc/php/conf.d/opcache.ini \
    && echo "opcache.max_accelerated_files=20000" >> /usr/local/etc/php/conf.d/opcache.ini

COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www

COPY . /var/www

RUN composer install --no-dev --optimize-autoloader --no-interaction

RUN chown -R www-data:www-data /var/www

EXPOSE 9000
CMD ["php-fpm"]

Key Takeaways

  • Consistency - Docker ensures identical environments everywhere
  • Docker Compose - Manage multi-container setups easily
  • Volume mounting - Enable live code reloading during development
  • Separate configs - Use different Dockerfiles for dev and production
  • OPcache - Enable in production for better performance
  • Security - Run containers as non-root user in production

Conclusion

Docker transforms PHP development by providing consistent, isolated environments that mirror production. Start with the basic setup I've shown here, then gradually add services like Redis, Elasticsearch, or queue workers as your application grows.

The initial setup takes time, but the benefits—consistent environments, easy onboarding, and production parity—make it worthwhile for any serious PHP project.
